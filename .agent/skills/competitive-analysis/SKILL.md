# Competitive Analysis Skill

> Expert at competitive analysis — landscape mapping, feature comparison, positioning, and strategic implications.

## Competitive Landscape Mapping

### Identify the Competitive Set

| Level | Definition | Example |
|---|---|---|
| **Direct** | Same problem, same users, same way | Products customers actively evaluate against you |
| **Indirect** | Same problem, different approach | Spreadsheets vs dedicated PM tool |
| **Adjacent** | Don't compete today, could tomorrow | Larger platforms that could add your feature |
| **Substitute** | Entirely different way to solve need | Hiring a person instead of buying software |

### Landscape Map Axes
Position competitors on meaningful dimensions:
- Breadth vs depth (suite vs point solution)
- SMB vs enterprise (market segment)
- Self-serve vs sales-led (GTM)
- Simple vs powerful (complexity)
- Horizontal vs vertical (general vs industry-specific)

Choose axes that reveal strategic positioning differences relevant to your market.

### Monitoring
Track over time: product launches, pricing changes, funding rounds, key hires, job postings (signal direction), customer wins/losses, analyst coverage, partnerships.

## Feature Comparison Matrices

### Building a Comparison
1. **Define capability areas**: Group by what matters to BUYERS, not your architecture
2. **List specific capabilities**: Features to compare under each area
3. **Rate consistently**: Use same scale for all

### Rating Scale

| Rating | Meaning |
|---|---|
| **Strong** | Market-leading. Deep, well-executed. |
| **Adequate** | Functional. Gets job done, not differentiated. |
| **Weak** | Exists but limited. Significant gaps. |
| **Absent** | Not available. |

### Tips
- Rate based on real experience + reviews, not marketing claims
- Weight by what matters to TARGET customers, not total feature count
- Be honest where competitors are ahead — biased comparison = not credible
- Update regularly — comparisons get stale fast
- Include "why it matters" for each capability area

### Matrix Template
```
| Capability Area   | Our Product | Competitor A | Competitor B |
|-------------------|-------------|-------------|-------------|
| [Area 1]          |             |             |             |
|   [Feature 1.1]   | Strong      | Adequate    | Absent      |
|   [Feature 1.2]   | Adequate    | Strong      | Weak        |
| [Area 2]          |             |             |             |
|   [Feature 2.1]   | Strong      | Strong      | Adequate    |
```

## Positioning Analysis

### Extract Positioning
For each competitor, identify:
- **Target customer**: Who exactly are they serving?
- **Category claim**: What category do they own?
- **Differentiator**: What makes them different?
- **Value proposition**: What outcome do they promise?
- **Proof points**: Evidence (logos, metrics, awards, case studies)

**Sources:** Homepage headline, app store descriptions, review sites, sales decks, earnings calls.

### Find Gaps and Opportunities
- **Unclaimed positions**: Value props no competitor owns that matter to buyers
- **Crowded positions**: Claims everyone makes that lost meaning
- **Emerging positions**: New value props from market changes (AI, remote, compliance)
- **Vulnerable positions**: Claims competitors can't fully deliver

## Win/Loss Analysis

### Data Sources (ranked by value)
1. Customer interviews shortly after decision (most valuable, least biased)
2. Churned customer surveys/exit interviews
3. CRM notes from sales team (available fast, but biased)
4. Prospect surveys for lost deals

### Key Questions
**For wins:** Why us over alternatives? What almost made you choose someone else? What would we need to lose?
**For losses:** What did you choose? Where did we fall short? Would you reconsider? Under what conditions?

### Common Patterns
- **Feature gap**: Competitor has capability that's a dealbreaker
- **Integration advantage**: Competitor fits existing tool stack
- **Pricing structure**: Different model fits better (per-seat vs usage-based)
- **Incumbent advantage**: Switching cost too high
- **Sales execution**: Better demo, faster response, more relevant case studies
- **Brand/trust**: Buyer chooses safer/more known option

## Market Trend Analysis

### For Each Trend, Assess:
1. **What's changing?** Describe specifically
2. **Why now?** Technology, regulation, behavior, economics?
3. **Who's affected?** Which customer segments?
4. **Timeline?** Happening now, 1-2 years, 3-5 years?
5. **Implication for us?** How should this influence strategy?
6. **Competitor response?** How are they reacting?

### Signal vs Noise
- **Signals**: Trends backed by behavioral data, investment, regulation, customer demand
- **Noise**: Only media hype, conference buzz, competitor announcements without traction
- Test against YOUR customer data: are YOUR customers experiencing this?

### Strategic Response Options
- **Lead**: Invest early, define the category. High risk, high reward.
- **Fast follow**: Wait for demand signals, then move quickly.
- **Monitor**: Track but don't invest yet. Set triggers.
- **Ignore**: Explicitly decide not relevant. Document why.

## Strategic Implications (Most Important Section)

Every competitive analysis must end with "so what":
- What should we build, accelerate, or deprioritize?
- Where should we differentiate vs achieve parity?
- How should we adjust positioning or messaging?
- What should we monitor going forward?

> Be honest about competitor strengths. Dismissing competitors makes the analysis useless. Focus on what matters to CUSTOMERS, not product teams.
