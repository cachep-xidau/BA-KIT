# Stakeholder Analysis

Identify, analyze, and plan engagement with project stakeholders.

## Identification

**Categories:**
- Internal: Executives, managers, employees
- External: Customers, vendors, regulators
- Project: Sponsors, PMs, BAs, devs, QA, users

**Discovery:** Org charts, interviews, process ownership, historical docs

## RACI Matrix

**R**esponsible: Does work | **A**ccountable: Decides (1 per task)
**C**onsulted: Input before | **I**nformed: Notified after

| Task | PO | PM | Dev | QA | BA |
|------|----|----|-----|----|-----|
| Define reqs | C | I | C | C | R,A |
| Design | I | I | R,A | C | C |
| Develop | I | I | R,A | I | C |
| Test | I | I | C | R,A | C |

## Power/Interest Grid

```
High Interest: Keep Satisfied | Manage Closely (Key)
Low Interest:  Monitor        | Keep Informed
               Low Power      | High Power
```

**High Power + High Interest**: Engage closely, key decisions
**High Power + Low Interest**: Keep satisfied, don't overwhelm
**Low Power + High Interest**: Keep informed, good ambassadors
**Low Power + Low Interest**: Monitor, minimal effort

## Conflict Resolution
1. Identify root cause
2. Understand all perspectives
3. Find common ground
4. Escalate if needed (with data)
5. Document decision rationale

## Meeting Types
**Kickoff**: Intro, objectives, roles, timeline
**Status**: Progress, blockers, decisions, next steps
**Review**: Demo, feedback, approvals
**Retro**: What worked, improvements, actions
